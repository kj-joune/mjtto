# 행운권 자동발행 및 문자발송 관리 기능 기획안

## 1. 문서 개요

- 문서명: 행운권 자동발행 및 문자발송 관리 기능 기획안
- 작성 목적: 행운권 자동발행 기능의 도입 필요성과 운영 구조를 정리하고, 구현 전 검토 및 의사결정 자료로 활용
- 적용 대상: 관리자 웹 시스템
- 작성 기준: 현행 발권/문자 시스템 구조를 최대한 재사용하는 방향

## 2. 추진 배경

현재 시스템은 지점 관리자가 직접 경품권을 발권하고, 당첨자에 대한 문자 발송 및 후속 처리를 수동으로 운영하는 구조이다.  
이 방식은 일반 운영에는 적합하나, 이벤트성 또는 반복성 행운권 발행 업무에는 다음과 같은 한계가 있다.

- 특정 요일/시간에 반복적으로 발행해야 하는 업무를 수동으로 처리해야 함
- 대상자 수가 많아질수록 누락, 중복, 오발송 가능성이 증가함
- 지점별 행사 운영 시 발행 이력과 실행 결과를 체계적으로 추적하기 어려움
- 문자 실패, 지점 코드 오류, 대상자 중복 등 예외 상황을 개별 관리하기 어려움

따라서 사전에 발행 조건과 수신자 목록을 등록해 두고, 지정된 시각에 자동으로 발권 및 문자 발송을 수행하는 관리 기능이 필요하다.

## 3. 추진 목적

본 기능의 도입 목적은 다음과 같다.

- 반복적인 행운권 발행 업무 자동화
- 지점 단위 이벤트 운영 효율 향상
- 발행 대상, 실행 결과, 실패 이력의 체계적 관리
- 기존 발권 및 문자 시스템과의 연동을 통한 안정적 확장
- 향후 통계, 재처리, 정산 연계 기능의 기반 확보

## 4. 현행 시스템 분석

현행 시스템에는 이미 다음과 같은 기반 기능이 존재한다.

- 지점 기준 발권 처리 구조
- 발권번호 및 티켓번호 생성 로직
- 발권 상세 이력 저장 구조
- 문자 발송 함수 및 문자 로그 저장 기능
- 제휴사/지점/관리자 권한 체계

즉, 본 기능은 완전 신규 시스템 구축보다는 기존 발권 엔진 위에 `예약 발행 관리 레이어`를 추가하는 방식이 적절하다.

## 5. 기능 정의

행운권 자동발행 기능은 관리자가 발행리스트를 등록하고, 그 발행리스트에 연결된 수신자 목록을 기준으로 지정된 요일 및 시간에 자동 발행을 수행하는 기능이다.

자동 실행 시 시스템은 다음 절차를 수행한다.

1. 현재 일시와 일치하는 발행리스트 조회
2. 해당 발행리스트의 유효한 수신자 목록 조회
3. 수신자별 발행지점을 확인하여 자동 발권 수행
4. 발권 완료 후 문자 발송 수행
5. 발행 결과 및 문자 결과를 실행 로그에 저장
6. 실패 건은 사유를 남기고 결과 화면에서 재확인 가능하도록 처리

## 6. 범위

### 6.1 포함 범위

- 발행리스트별 수신자 목록 관리
- 지정 시각 자동 발권
- 문자 자동 발송
- 실행 결과 로그 관리
- 실패 건 확인 및 재처리 검토 기반

### 6.2 제외 범위

- 즉시 개발 및 운영 배포
- 외부 고객용 화면 제공
- 정산 시스템 직접 연동
- 대규모 통계/리포트 기능

## 7. 업무 흐름

4. 지점 코드 및 전화번호 검증
5. 저장 후 실행 대기 상태 유지

### 7.2 시스템 자동 실행 흐름

3. 실행 대상 수신자 목록 추출
4. 지점 기준 자동 발권 수행

## 10. 정책 검토 사항

구현 전 반드시 확정해야 할 정책은 다음과 같다.

### 10.1 발행 방식

#### 선택안

- 1회형: 기간 내 조건이 맞는 첫 실행 1회만 발행
- 반복형: 기간 내 지정 시간마다 반복 발행

#### 권장안

`1회형` 우선 적용

#### 사유

- 중복 발행 위험 감소
- 운영 관리 단순화
- 고객 민원 가능성 감소
- 결과 추적이 용이함

### 10.2 문자 실패 처리 기준

#### 선택안

- 문자 실패 시 발권도 취소
- 발권은 유지하고 문자 실패만 기록

#### 권장안

`발권 유지 + 문자 실패 기록`

#### 사유

- 발권 취소 로직은 이력 관리가 복잡해짐
- 재발송 방식이 운영상 현실적임
- 실패 원인을 분리 관리하기 쉬움

### 10.3 지점 식별 방식

#### 선택안

- 지점코드만 저장
- 등록 시점에 지점코드를 검증해 확정 지점 ID 저장

#### 권장안

`등록 시점 확정 저장`

#### 사유

- 실행 시 오류 감소
- 세부 화면에서 검증 상태를 명확히 표시 가능
- 지점 변경/오입력 관리가 쉬움

### 10.4 중복 발행 기준

#### 검토 항목

- 동일 고객 중복 등록 허용 여부
- 동일 발행리스트 내 중복 허용 여부
- 동일 슬롯 재실행 시 중복 발행 허용 여부

#### 권장안

동일 발행리스트, 동일 대상자, 동일 실행 슬롯 기준 중복 차단

## 11. 운영상 유의사항

- 자동발권 건은 일반 수동발권과 구분되는 메모가 필요함
- 발권 실패와 문자 실패를 반드시 분리 기록해야 함
- 수신자 등록 단계에서 전화번호 정규화가 필요함
- 지점코드 오류는 실행 전 등록 단계에서 최대한 차단해야 함
- 실행 이력은 추후 정산 및 운영 리포트와 연결될 가능성을 고려해야 함

## 12. 시스템 연계 방향

본 기능은 다음 기존 기능과 연계하는 방향이 적절하다.

- 기존 지점 기준 발권 함수 재사용
- 기존 문자 발송 함수 재사용
- 기존 문자 로그 저장 구조 재사용
- 기존 관리자 권한 체계 활용

이를 통해 신규 개발 범위를 줄이고 운영 안정성을 높일 수 있다.

## 14. 결론

행운권 자동발행 및 문자발송 관리 기능은 현행 시스템 구조를 활용하여 비교적 안정적으로 확장 가능한 기능이다.  
특히 발행리스트, 수신자 목록, 실행결과의 3단 구조로 설계할 경우 운영자 입장에서 기능 흐름이 명확하고, 향후 유지보수 및 확장성도 확보할 수 있다.

- 화면 3종 구성

  - 발행 마스터
  - 수신자 목록
  - 실행 로그
- 운영 정책
  - 기본 1회형
  - 문자 실패 시 발권 유지
  - 등록 시 지점코드 검증 및 확정 저장
